This is a demo problem on how to use the new drag-drop-elements.

drag-drop-elements works like pl-multiple-choice, in which the options are nested within the drag-drop-elements, and the correctness of the options are specified with an correct attribute.

Instructors can specify the options to be shuffled by specifying the shuffle-options attribute to be true within the drag-drop-elements tag. Much like pl-multiple-choice, you also need to specify an answers-name.

As shown in this demo, it is possible to use multiple drag-drop-elements in a single question. Be sure all of the elements have different answers-name attribute if you do so.

One of the most powerful features of this custom element is the permutation-mode the element offers. permutation-mode allows 3 modes, any, html-order, and ranked:

  • any awards full credits to student submissions that contains all MCQ options with the correct="true" attribute, and partial credit for each correct MCQ options that are present. For illustration, a submission that has 4 correct answers when there are 5 true answers will receive 80%.
  • html-order awards full credits only if the following both holds: the student submission contains all MCQ options with the correct="true" attribute, and the student submission ordering matches the ordering of the pl-answers in the question HTML page. Consequently, html-order does not award partial credits.
  • ranked requires the use of an additional attribute called ranking in the pl-answer tags for MCQ options that are correct. The ranking attribute allows you to specify the correct ordering of MCQ options using positive numbers (excluding 0). Notably, multiple answers can have identical ranking. For answers with identical ranking, the grader will consider any permutations for those answers to be correct. This is useful for coding questions. The grader will award credit for each correctly ordered MCQ option, until there are no more options to grade, or an incorrectly ordered MCQ answer/incorrect answer is encountered.
