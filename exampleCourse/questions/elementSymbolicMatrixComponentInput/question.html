Differences from the normal matrix input:

The symbolic matrix input combines the symbolic expression parsing of the symbolic input field with the layout and some of the UI and floating point features of the matrix component input. (The ability to accept floating point is actually different from the normal symbolic input too!)

The symbolic matrix input has a show-correct-answer attribute in addition to allow-feedback. So, it's possible to allow inline feedback of partially-credit responses but hide the full correct answer display. Below, the $\mathbf{b}$ matrix input has ths set to false.

Other attributes include allow-complex which allows or disallows the $i$ variable to serve as an imaginary unit, as with the individual (legacy) symbolic input element. The variables attribute whitelists variable name strings. Below, we allow variables='x, y'. The "?" button on the input element has additional notes about how to correctly enter symbolic expressions.

As with the normal matrix component input, answers-name is what can be used to index into Python objects in the question code. For example, if answers-name='outA', then:

```python data["correct_answers"]["outA"] = pl.to_json(A_solution) ```

where the A_solution variable is a sympy matrix containing the instructor's intended solution.

You can also specify placeholder values, input that will appear initially instead of empty fields. These can be accessed through the placeholder param in the code after initializing it, further indexed by answers_name_1 ... _99, etc. For example:

```python # Initialize placeholder object data["params"]["placeholder"] = {} # Add placeholder values, counting from 1, and going across rows: data["params"]["placeholder"]["outA_1"] = "0" data["params"]["placeholder"]["outA_2"] = "0" # ... data["params"]["placeholder"]["outA_16"] = "0" data["params"]["placeholder"]["outb_1"] = "0" # ...```
